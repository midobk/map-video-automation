# Assets and audio

## Asset folder conventions

All render assets live under `apps/remotion-studio/public/` and are referenced
with `staticFile()`. Conventions:

- `public/sfx/` — shared UI sound effects (generated by `pnpm sfx:generate`).
- `public/<project>/` — per-project static assets (logos, screenshots,
  voiceover). The PR 1A fixture uses no project assets.

Per-project folders keep unrelated videos isolated, matching the donor's
`public/<project>/` convention without importing any donor assets.

## Required vs optional assets

An `AssetDeclaration` declares whether an asset is `required` or optional and
its expected `type` (`image`, `audio`, `video`, `font`).

- **Required assets must exist.** A missing required asset fails explicitly
  rather than silently producing an incomplete video. Required narration always
  errors, in every environment.
- **Optional assets may be absent.** A missing optional asset is ignored in a
  production render and warned about in Studio.

## Static-file rules

Use `resolveStaticPath(path, publicDir)` for any path that comes from a plan or
external input. It:

- rejects absolute paths and Windows drive paths,
- rejects path traversal (`..`, escaping `public/`),
- normalizes to POSIX separators,
- validates the extension against the declared asset type,
- rejects duplicate paths in a manifest.

Never accept an unvalidated string as a static path. Never use machine-specific
absolute paths in a plan.

## Font loading rules

Fonts are loaded through `@remotion/google-fonts`, which bundles each family as
its own subpath and blocks rendering until the font is ready. `resolveFontFamily`
only accepts families registered in `ALLOWED_FONTS`. Do not accept arbitrary
family strings — that would allow remote runtime downloads of un-vetted fonts.
To add a family, import its subpath and register it in `ALLOWED_FONTS`.

## Audio rules

`MissingAssetBehavior` is `'ignore' | 'warn' | 'error'`. Required defaults:

| Environment | Required | Optional (SFX) | Required narration |
|---|---|---|---|
| Studio preview | error | warn | error |
| Production render | error | ignore | error |
| Tests | error | error | error |

`<OptionalAudio>` is the `<Audio>` wrapper. In Studio it probes the static-files
list before mounting `<Audio>`, so a missing file never triggers the media-bunny
404. In a production render there is no Studio static-files API, so it renders
audio unconditionally; required assets are validated before rendering, so a
missing required asset fails loudly through Remotion itself.

The Studio file-existence probe is injected through `AudioConfigProvider` so the
renderer never imports `@remotion/studio` directly. `@remotion/studio` is an
explicit, optional concern of the Studio app, not a hidden renderer dependency.
